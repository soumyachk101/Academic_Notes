Group A — Multiple Choice Questions

1. (i) Which of the following algorithm design techniques is used in quick sort algorithm?

Answer: c) Divide and Conquer (by partitioning the array around a pivot and sorting subarrays recursively).

1. (ii) Which of the following algorithms solves the All-Pair Shortest Path problem?

Answer: b) Floyd's Warshall's (which is a dynamic programming algorithm with complexity \(O(V^3)\)).

1. (iii) Travelling Salesman problem is:

Answer: a) NP Hard (specifically the optimization version is NP-Hard, while the decision version is NP-Complete).

1. (iv) Complexity of the recurrence relation T(n) = 8T(n/2) + n² is:

Answer: d) O(n³).

Proof: Using Master's Theorem with \(a=8, b=2, f(n)=n^2\).
\(n^{\log_b a} = n^{\log_2 8} = n^3\).
Since \(f(n) = O(n^{3 - \epsilon})\) for \(\epsilon=1\), this matches Case 1 of Master's Theorem, yielding \(T(n) = \Theta(n^3)\).

1. (v) The minimum number of colours needed to colour a graph having n > 3 vertices and 2 edges is:

Answer: a) 2 (since the graph has at most two disjoint edges, its maximum degree is small, and there are no odd cycles. A simple 2-coloring is always feasible).

1. (vi) The fractional Knapsack problem can be solved by using:

Answer: a) Greedy method.

1. (vii) Time complexity of Binary Search Algorithm on n items is:

Answer: d) O(log n).

1. (viii) The time factor when determining the efficiency of an algorithm is measured by:

Answer: b) counting number of key operations (this provides a hardware-independent metric of complexity).

1. (ix) The tight bound for building max heap is:

Answer: a) O(n).

1. (x) BFS of a graph G = (V, E) has running time:

Answer: a) O(|V| + |E|).

Group B — Short Answer Type Questions

2. Find the best and worst case complexity of Quick sort.

  • Best Case (\(O(n \log n)\)): Occurs when partitions split the array into exactly equal halves. Recurrence: \(T(n) = 2T(n/2) + \Theta(n)\). Applying Case 2 of Master's Theorem yields \(T(n) = \Theta(n \log n)\).
  • Worst Case (\(O(n^2)\)): Occurs when partitions split the array in a highly unbalanced way (e.g. \(0\) and \(n-1\) elements), which happens when the array is already sorted and we pick the last/first element as pivot. Recurrence: \(T(n) = T(n-1) + \Theta(n)\), which expands to \(\Theta(n^2)\).

3. State master's theorem and find the time complexity for the following recurrence: T(n) = 2T(n^(1/2)) + log n

Master's Theorem Definition: For recurrences of the form \(T(n) = a T(n/b) + f(n)\):

  1. If \(f(n) = O(n^{\log_b a - \epsilon})\), then \(T(n) = \Theta(n^{\log_b a})\).
  2. If \(f(n) = \Theta(n^{\log_b a} \log^k n)\), then \(T(n) = \Theta(n^{\log_b a} \log^{k+1} n)\).
  3. If \(f(n) = \Omega(n^{\log_b a + \epsilon})\) and regularity condition holds, then \(T(n) = \Theta(f(n))\).

Solving \(T(n) = 2T(n^{1/2}) + \log n\):

We use a change of variables to convert this into Master Theorem form.
Let \(m = \log_2 n \implies n = 2^m\). This means \(n^{1/2} = (2^m)^{1/2} = 2^{m/2}\).
Substituting these: \[T(2^m) = 2T(2^{m/2}) + m\] Let \(S(m) = T(2^m)\). The recurrence becomes: \[S(m) = 2S(m/2) + m\] Now we apply Master's Theorem to \(S(m)\) with \(a=2, b=2, f(m)=m\):
- \(m^{\log_b a} = m^{\log_2 2} = m^1 = m\).
- Since \(f(m) = \Theta(m)\), this matches Case 2 (\(k=0\)).
- Therefore, \(S(m) = \Theta(m \log_2 m)\).
Substituting back \(m = \log_2 n\): \[T(n) = \Theta(\log n \cdot \log(\log n))\]

Complexity: \(\Theta(\log n \log(\log n))\)

4. Given the weight vector (2, 3, 5, 7, 1, 4, 1) and the profit vector (10, 5, 15, 7, 6, 18, 3) and a Knapsack of capacity 15. Find at least three feasible solutions including optimal one for the knapsack problem of seven objects.

We solve the 0-1 Knapsack problem where capacity \(M = 15\) and items are represented by index, weight, and profit.

Feasible Solutions: Any subset of items whose total weight does not exceed 15 is a feasible solution.

  1. Feasible Solution 1 (Greedy by Ratio):
    Sort by \(P_i / W_i\) ratio: Item 5 (ratio 6.0) \(\to\) Item 1 (5.0) \(\to\) Item 6 (4.5) \(\to\) Item 3 (3.0) \(\to\) Item 7 (3.0) \(\to\) Item 2 (1.67) \(\to\) Item 4 (1.0).
    Selecting in order of ratio: Item 5, Item 1, Item 6, Item 3, Item 7.
    Total Weight = 1 + 2 + 4 + 5 + 1 = 13 kg (\(\le 15\)).
    Total Profit = 6 + 10 + 18 + 15 + 3 = 52.
  2. Feasible Solution 2 (Greedy by Profit):
    Sort by profit: Item 6 (18) \(\to\) Item 3 (15) \(\to\) Item 1 (10) \(\to\) Item 4 (7) \(\to\) Item 5 (6) \(\to\) Item 2 (5) \(\to\) Item 7 (3).
    Selecting in order of profit: Item 6, Item 3, Item 1, Item 5, Item 2.
    Total Weight = 4 + 5 + 2 + 1 + 3 = 15 kg (\(\le 15\)).
    Total Profit = 18 + 15 + 10 + 6 + 5 = 54.
  3. Feasible Solution 3 (Optimal Solution):
    By applying dynamic programming, the maximum achievable profit is obtained by selecting items 1, 2, 3, 5, and 6.
    Total Weight = 2 + 3 + 5 + 1 + 4 = 15 kg (\(\le 15\)).
    Total Profit = 10 + 5 + 15 + 6 + 18 = 54.
Optimal subset is {Item 1, Item 2, Item 3, Item 5, Item 6} with max profit 54.

5. Solve the following recurrence relation using generating function: a_n = 6a_(n-1) - 11a_(n-2) + 6a_(n-3) for n >= 3 with initial condition a_0 = 1, a_1 = -1 and a_2 = 1.

The relation is \(a_n - 6a_{n-1} + 11a_{n-2} - 6a_{n-3} = 0\). The characteristic equation is:

\[r^3 - 6r^2 + 11r - 6 = 0\]

We solve the roots of the characteristic equation:

  • Try \(r = 1\): \(1^3 - 6(1^2) + 11(1) - 6 = 0\). Thus, \((r-1)\) is a root.
  • Dividing: \((r-1)(r^2 - 5r + 6) = (r-1)(r-2)(r-3) = 0\).
  • The roots are \(r_1=1, r_2=2, r_3=3\).

The general solution is of the form: \(a_n = C_1 \cdot 1^n + C_2 \cdot 2^n + C_3 \cdot 3^n = C_1 + C_2 2^n + C_3 3^n\).

We solve the constants using initial conditions:

  1. For \(n=0\): \(C_1 + C_2 + C_3 = 1 \quad \text{--- (Eq 1)}\)
  2. For \(n=1\): \(C_1 + 2C_2 + 3C_3 = -1 \quad \text{--- (Eq 2)}\)
  3. For \(n=2\): \(C_1 + 4C_2 + 9C_3 = 1 \quad \text{--- (Eq 3)}\)

Subtracting Eq 1 from Eq 2: \(C_2 + 2C_3 = -2 \quad \text{--- (Eq 4)}\)

Subtracting Eq 2 from Eq 3: \(2C_2 + 6C_3 = 2 \implies C_2 + 3C_3 = 1 \quad \text{--- (Eq 5)}\)

Subtracting Eq 4 from Eq 5: \(C_3 = 3\).

Substituting \(C_3 = 3\) in Eq 4: \(C_2 + 6 = -2 \implies C_2 = -8\).

Substituting both in Eq 1: \(C_1 - 8 + 3 = 1 \implies C_1 = 6\).

Final Solution: \(a_n = 6 - 8 \cdot 2^n + 3 \cdot 3^n\)

6. Discuss Job Sequencing with Deadlines with an example.

Job Sequencing with Deadlines: We are given a set of \(n\) jobs, where each job \(i\) has a deadline \(d_i \ge 1\) and a profit \(p_i > 0\). Each job takes 1 unit of time to execute on a single processor, and profit is earned only if the job is completed within its deadline. The goal is to find a schedule that maximizes the total profit.

Example: Consider 4 jobs with profits and deadlines:

  • \(J_1\): Profit = 100, Deadline = 2
  • \(J_2\): Profit = 10, Deadline = 1
  • \(J_3\): Profit = 15, Deadline = 2
  • \(J_4\): Profit = 27, Deadline = 1

Step-by-step Greedy Selection:

  1. Sort jobs by profit descending: \(J_1(100) \to J_4(27) \to J_3(15) \to J_2(10)\).
  2. Maximum deadline is 2. Create time slots: Slot 1 (0-1) and Slot 2 (1-2).
  3. Pick \(J_1\) (Deadline 2): Place in latest free slot before deadline \(\to\) Slot 2 (1-2).
  4. Pick \(J_4\) (Deadline 1): Place in latest free slot before deadline \(\to\) Slot 1 (0-1).
  5. Pick \(J_3\) (Deadline 2): Latest slot before 2 is Slot 2 (full). Slot 1 is also full. Reject.
  6. Pick \(J_2\) (Deadline 1): Slots full. Reject.
Optimal Sequence: J4 followed by J1 (or J1 and J4)
Max Profit: 27 + 100 = 127

Group C — Long Answer Type Questions

7. (a) Matrix Chain Multiplication Dynamic Programming algorithm and explanation.

The matrix chain multiplication algorithm computes the minimum number of multiplications to multiply a chain of \(n\) matrices.

Matrix-Chain-Order(p):
    n = length(p) - 1
    m[1..n, 1..n] = table, s[1..n-1, 2..n] = table
    For i = 1 to n:
       m[i, i] = 0
    For L = 2 to n: // L is chain length
       For i = 1 to n - L + 1:
          j = i + L - 1
          m[i, j] = Infinity
          For k = i to j - 1:
             q = m[i, k] + m[k+1, j] + p[i-1]*p[k]*p[j]
             If q < m[i, j]:
                m[i, j] = q
                s[i, j] = k
    Return m and s

Explanation: The algorithm computes the values bottom-up by increasing chain lengths \(L\). The auxiliary table \(s[i, j]\) records the optimal split index \(k\) that achieves the minimum multiplication cost, allowing us to reconstruct the exact parenthesized matrix sequence.

(b) Critically comment on "Greedy strategy does not work for the 0-1 knapsack problem for all time".

In a 0-1 Knapsack, fractional items are not permitted. If we pick items based on the highest profit-to-weight ratio, we may leave empty space in the knapsack that cannot be filled, leading to suboptimal solutions compared to a dynamic programming combination that fills the capacity more effectively. (See countermeasures and example in Q13 of the Important Questions bank).

(c) What do you mean by non-deterministic algorithms?

A non-deterministic algorithm is an algorithm that can exhibit different behaviors on different runs for the same input. It contains operations with multiple possible outcomes (non-deterministic choices). In complexity theory, a non-deterministic machine can run all paths in parallel, solving problems in polynomial time if any path reaches a valid solution (defining the class NP).

8. (a) State the general Knapsack problem. Write a greedy algorithm for this problem and derive its time complexity.

General Knapsack Problem: Given a set of \(n\) items, each with weight \(w_i\) and profit \(p_i\), and a knapsack capacity \(W\), we want to choose a subset of items to maximize the total profit such that the total weight is \(\le W\). If items can be divided, it is the Fractional Knapsack; if not, it is the 0-1 Knapsack.

Greedy Algorithm for Fractional Knapsack:

Greedy-Fractional-Knapsack(W, p[], w[], n):
    1. Calculate ratios r[i] = p[i]/w[i] for all items.
    2. Sort items by r[i] in descending order.
    3. Initialize current_weight = 0, total_profit = 0.
    4. For i = 1 to n:
          If current_weight + w[i] <= W:
             current_weight += w[i]
             total_profit += p[i]
          Else:
             remain = W - current_weight
             total_profit += p[i] * (remain / w[i])
             Break
    5. Return total_profit

Complexity derivation: Calculating ratios takes \(O(n)\) time. Sorting the items takes \(O(n \log n)\) time. The selection loop takes \(O(n)\) time. Overall time complexity is \(O(n \log n)\).

9. (a) Describe the Depth first search algorithm for a given graph and explain its time complexity.

DFS Algorithm: Traverses a graph by visiting a vertex, and recursively exploring all its unvisited adjacent vertices before backtracking.

DFS(G):
    For each vertex u in G.V:
       u.color = White
    time = 0
    For each vertex u in G.V:
       If u.color == White:
          DFS-Visit(G, u)

DFS-Visit(G, u):
    time = time + 1
    u.d = time
    u.color = Gray
    For each v in G.adj[u]:
       If v.color == White:
          DFS-Visit(G, v)
    u.color = Black
    time = time + 1
    u.f = time

Time Complexity: Every vertex is visited exactly once, taking \(O(V)\) total time. The adjacency lists of all vertices are scanned exactly once, taking \(O(E)\) time. Thus, the total complexity is \(\Theta(V + E)\).

(b) Write the algorithm for graph colouring problem. (c) Apply backtracking to solve 3-colouring.

(b) Backtracking Graph Coloring Algorithm:

GraphColoring(vertex, color_assignment, m, G):
    If vertex == G.V:
       Return True // Success
    For c = 1 to m:
       If IsSafe(vertex, c, color_assignment, G):
          color_assignment[vertex] = c
          If GraphColoring(vertex + 1, color_assignment, m, G):
             Return True
          color_assignment[vertex] = 0 // Backtrack
    Return False

(c) Applying 3-Coloring on the Graph:
Let the graph vertices be labeled as \(V = \{1, 2, 3, 4, 5\}\). Colors are \(\{C_1, C_2, C_3\}\).
1. Assign vertex 1 color \(C_1\).
2. Vertex 2 is adjacent to 1. Assign it color \(C_2\).
3. Vertex 3 is adjacent to 1 and 2. Assign it color \(C_3\).
4. Vertex 4 is adjacent to 2 and 3. Assign it color \(C_1\).
5. Vertex 5 is adjacent to 4 and 1. Assign it color \(C_2\) (safe since it is not adjacent to 2).
This demonstrates a valid 3-coloring path with no conflicts.

11. Write short notes on any three of the following: a) Vertex Cover, b) Recursion Tree, c) Heap Creation, d) Approximation schemes, e) Asymptotic notation.

a) Vertex Cover: A subset of vertices \(V' \subseteq V\) in an undirected graph such that every edge in the graph has at least one endpoint in \(V'\). The decision version is NP-Complete; the optimization version seeks the minimum vertex cover.

b) Recursion Tree: A visual representation of a recurrence relation. Nodes represent the cost of solving subproblems at different levels of recursion. Summing the costs across all levels yields the total time complexity.

c) Heap Creation (Build-Max-Heap): Converts an unordered array into a Max-Heap. It starts at the last non-leaf node (\(\lfloor n/2 \rfloor\)) and runs Max-Heapify bottom-up. The time complexity is bounded by \(O(n)\) (not \(O(n \log n)\)) because the cost of heapifying decreases exponentially with height.
