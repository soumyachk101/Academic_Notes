Group A — Very Short Answer Type Questions

1. (i) What is the appropriate data structure for Prim's Minimum Spanning Tree algorithm?

Answer: Min-Priority Queue (typically implemented using a Binary Heap or a Fibonacci Heap for optimal performance). It is used to quickly extract the vertex with the minimum key value among the unvisited vertices.

1. (ii) What is represented by the asymptotic notation O(1)?

Answer: Constant Time Complexity. It indicates that the execution time of the algorithm remains constant and does not grow or change with the size of the input data \(n\).

1. (iii) Time complexity for recurrence relation T(n) = 2T(n/2) + n is:

Answer: \(O(n \log n)\). This matches Case 2 of Master's Theorem where \(a=2, b=2, k=0 \implies n^{\log_2 2} = n\), resulting in \(T(n) = \Theta(n \log n)\).

1. (iv) The notation \(\Omega(n)\) is the formal way to express the ______ bound of an algorithm's running time.

Answer: asymptotic lower (representing the best-case growth rate).

1. (v) Fractional knapsack problem is solved most efficiently by which algorithm design technique?

Answer: Greedy method (sorting by value-to-weight ratio and picking items greedily).

1. (vi) What is the appropriate data structure for Depth First Search algorithm?

Answer: Stack (either using the system call stack via recursion, or an explicit LIFO Stack data structure).

1. (vii) Which algorithm is used to solve a maximum flow problem?

Answer: Ford-Fulkerson Algorithm (or its polynomial implementation, the Edmonds-Karp Algorithm using BFS).

1. (viii) How many solutions are there for 8 queens problem on 8*8 board?

Answer: 92 total solutions (which reduce to 12 unique solutions under rotational and reflective symmetry).

1. (ix) Time complexity of Heap sort with n items is:

Answer: \(O(n \log n)\) in all cases (best, worst, and average cases).

1. (x) The 0-1 Knapsack problem can be solved using Greedy algorithm - state True or False.

Answer: False. Greedy algorithms do not guarantee an optimal solution for 0-1 Knapsack because items cannot be divided; Dynamic Programming or Branch and Bound must be used.

1. (xi) Which algorithm is used to solve the single source shortest path problem in a graph with negative edge weights?

Answer: Bellman-Ford Algorithm (can handle negative edge weights and detect negative cycles).

1. (xii) Given items as {value, weight} pairs {{40, 20}, {30, 10}, {20, 5}}. The capacity of knapsack is 20. Find the maximum value output assuming items to be divisible.

Answer: 60.

Calculation:
1. Calculate value/weight ratios:

  • Item 1: \(40/20 = 2\)
  • Item 2: \(30/10 = 3\)
  • Item 3: \(20/5 = 4\)
2. Sort items by ratio: Item 3 \(\to\) Item 2 \(\to\) Item 1.
3. Load knapsack (Capacity = 20):
  • Take all of Item 3 (Weight = 5, Value = 20). Remaining capacity = 15.
  • Take all of Item 2 (Weight = 10, Value = 30). Remaining capacity = 5.
  • Take fraction of Item 1: \(\frac{5}{20} = 0.25\) of Item 1 (Weight = 5, Value = \(40 \times 0.25 = 10\)). Remaining capacity = 0.
Total Value = \(20 + 30 + 10 = 60\).

Group B — Short Answer Type Questions

2. What are the differences between Dijkstra's algorithm and Bellman-Ford algorithm? Compare their time complexities.

Dijkstra's Algorithm and Bellman-Ford Algorithm both solve the Single Source Shortest Path (SSSP) problem, but they target different constraints and use different strategies.

Feature Dijkstra's Algorithm Bellman-Ford Algorithm
Paradigm Greedy Approach (selects the nearest vertex next). Dynamic Programming (relaxes all edges iteratively).
Negative Edges Cannot handle negative edge weights. Can handle negative edge weights.
Cycle Detection Cannot detect negative cycles (will fail/loop). Can detect the presence of negative cycles.
Edge Relaxation Relaxes edges from a single visited vertex at a time. Relaxes all edges in the graph in every iteration.
Time Complexity \(O(E \log V)\) using a Binary Heap. \(O(V \cdot E)\) (slower than Dijkstra).

3. Analyze the time complexity of quick sort algorithm for best case and worst case using recurrence relations.

Quick Sort's performance is partition-dependent:

1. Best Case (Perfect Splits):

Occurs when the pivot splits the array into two equal halves of size \(n/2\) at each recursion level. Recurrence:

\[T(n) = 2T(n/2) + cn\]

Applying Case 2 of Master's Theorem (\(a=2, b=2, f(n)=cn \implies n^{\log_2 2} = n\)):

\(T(n) = \Theta(n \log n)\)

2. Worst Case (Skewed Splits):

Occurs when the pivot is consistently the smallest or largest element, leaving partitions of size \(0\) and \(n-1\). Recurrence:

\[T(n) = T(n-1) + T(0) + cn = T(n-1) + cn\]

Expanding the recurrence:

  • \(T(n) = T(n-1) + cn\)
  • \(T(n) = T(n-2) + c(n-1) + cn\)
  • \(T(n) = T(1) + c \sum_{i=2}^{n} i = T(1) + c \left[\frac{n(n+1)}{2} - 1\right]\)
\(T(n) = O(n^2)\)

4. Write the Floyd-Warshal Algorithm for all-pairs shortest path problem. What will be the time complexity?

Floyd-Warshall Algorithm: A dynamic programming approach to find the shortest path between all pairs of vertices. It systematically computes intermediate paths using vertices \(\{1..k\}\).

Floyd-Warshall(W, V):
    1. Let D^(0) = W (Initial weight matrix where D[i][j] = weight(i,j))
    2. For k = 1 to V:
          For i = 1 to V:
             For j = 1 to V:
                D^(k)[i][j] = min( D^(k-1)[i][j], D^(k-1)[i][k] + D^(k-1)[k][j] )
    3. Return D^(V)

Time Complexity: The three nested loops each run exactly \(V\) times, executing a constant-time comparison in each iteration. Thus, the total time complexity is \(\Theta(V^3)\).

5. For the given set of 5 items and the knapsack capacity of 10 kg, find the maximum profit. (Fractional knapsack, Wi={3,3,2,5,1}, Pi={10,15,10,12,8}).

Given capacity \(M = 10\) kg. We calculate the profit-to-weight ratios:

  1. Item 1: \(P_1 = 10, W_1 = 3 \implies ratio = 3.33\)
  2. Item 2: \(P_2 = 15, W_2 = 3 \implies ratio = 5.00\)
  3. Item 3: \(P_3 = 10, W_3 = 2 \implies ratio = 5.00\)
  4. Item 4: \(P_4 = 12, W_4 = 5 \implies ratio = 2.40\)
  5. Item 5: \(P_5 = 8, W_5 = 1 \implies ratio = 8.00\)

Sorting in descending order of ratio: Item 5 (8.0) \(\to\) Item 2 (5.0) \(\to\) Item 3 (5.0) \(\to\) Item 1 (3.33) \(\to\) Item 4 (2.40).

Greedy Selection:

  • Take all of Item 5 (Weight = 1 kg, Profit = 8). Remaining capacity = 9 kg.
  • Take all of Item 2 (Weight = 3 kg, Profit = 15). Remaining capacity = 6 kg.
  • Take all of Item 3 (Weight = 2 kg, Profit = 10). Remaining capacity = 4 kg.

6. Write short note on Randomized algorithms.

A Randomized Algorithm is an algorithm that uses random numbers to influence its decision-making logic. Instead of executing deterministically based on the input, it leverages a random source to obtain average-case behavior on worst-case inputs.

Randomized algorithms are broadly classified into two main types:

  1. Las Vegas Algorithms: These algorithms always produce the correct result, but their execution time is a random variable. The goal is to minimize the expected running time.
    Example: Randomized Quick Sort (choosing a random pivot avoids worst-case \(O(n^2)\) splits, guaranteeing expected \(O(n \log n)\) time).
  2. Monte Carlo Algorithms: These algorithms have a deterministic execution time, but their outputs have a small, bounded probability of being incorrect. The accuracy probability can be improved by running the algorithm multiple times.
    Example: Miller-Rabin Primality Test (checks if a number is prime in polynomial time with a tiny error probability), or Karger's Min-Cut Algorithm.

7. (a) Find the minimum number of multiplications required for Matrix Chain Multiplication: A(10×20), B(20×50), C(50×1), D(1×100).

Given matrices \(A(10 \times 20)\), \(B(20 \times 50)\), \(C(50 \times 1)\), and \(D(1 \times 100)\). The dimensions array is \(P = [10, 20, 50, 1, 100]\) for \(n = 4\) matrices. Recurrence: \(m[i, j] = \min \{m[i, k] + m[k+1, j] + P[i-1] \cdot P[k] \cdot P[j]\}\).

DP Calculation Table:

  1. Chain Length 1:
    \(m[1, 1] = m[2, 2] = m[3, 3] = m[4, 4] = 0\)
  2. Chain Length 2:
    • \(m[1, 2] = 10 \times 20 \times 50 = 10,000\) (Split at \(k=1\))
    • \(m[2, 3] = 20 \times 50 \times 1 = 1,000\) (Split at \(k=2\))
    • \(m[3, 4] = 50 \times 1 \times 100 = 5,000\) (Split at \(k=3\))
  3. Chain Length 3:
    • \(m[1, 3]\) (Multiplying \(A B C\)): Possible splits:
      • \(k = 1\): \(m[1, 1] + m[2, 3] + P[0] \cdot P[1] \cdot P[3] = 0 + 1000 + 10 \times 20 \times 1 = 1200\)
      • \(k = 2\): \(m[1, 2] + m[3, 3] + P[0] \cdot P[2] \cdot P[3] = 10000 + 0 + 10 \times 50 \times 1 = 10500\)
      Minimum is 1,200 at split \(k=1\).
    • \(m[2, 4]\) (Multiplying \(B C D\)): Possible splits:
      • \(k = 2\): \(m[2, 2] + m[3, 4] + P[1] \cdot P[2] \cdot P[4] = 0 + 5000 + 20 \times 50 \times 100 = 105,000\)
      • \(k = 3\): \(m[2, 3] + m[4, 4] + P[1] \cdot P[3] \cdot P[4] = 1000 + 0 + 20 \times 1 \times 100 = 3,000\)
      Minimum is 3,000 at split \(k=3\).
  4. Chain Length 4 (\(m[1, 4]\) - Multiplying \(A B C D\)): Possible splits:
    • \(k = 1\): \(m[1, 1] + m[2, 4] + P[0] \cdot P[1] \cdot P[4] = 0 + 3000 + 10 \times 20 \times 100 = 23,000\)
    • \(k = 2\): \(m[1, 2] + m[3, 4] + P[0] \cdot P[2] \cdot P[4] = 10000 + 5000 + 10 \times 50 \times 100 = 65,000\)
    • \(k = 3\): \(m[1, 3] + m[4, 4] + P[0] \cdot P[3] \cdot P[4] = 1200 + 0 + 10 \times 1 \times 100 = 2,200\)
    Minimum is 2,200 at split \(k=3\).
Minimum Scalar Multiplications: 2,200
Optimal Parenthesization: \(((A \times B) \times C) \times D\)

(b) Write the algorithm for single source shortest path (-ve edge weights allowed) [Bellman-Ford].

Bellman-Ford(Graph G, Source S):
    1. Initialize distances: dist[v] = Infinity for all v, dist[S] = 0
    2. Relax all edges |V| - 1 times:
       For i = 1 to |V| - 1:
          For each edge (u, v) in G with weight w:
             If dist[u] != Infinity and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
    3. Detect negative cycles:
       For each edge (u, v) in G with weight w:
          If dist[u] != Infinity and dist[u] + w < dist[v]:
             Error "Negative weight cycle detected!"
    4. Return dist

(c) Discuss the advantages and drawbacks of Dynamic Programming.

Advantages: Guarantees global optimality; avoids redundant computations by storing intermediate subproblem states; simple recursive logic formalized into clean tables.

Drawbacks: High space complexity to maintain DP tables; designing transition relations and states is mathematically complex; strict requirements of optimal substructure and overlapping subproblems.

8. (a) What are the features of any Algorithm?

An algorithm must satisfy Donald Knuth's five primary properties:

  1. Input: Accepts zero or more values externally.
  2. Output: Produces at least one output matching the goal.
  3. Definiteness: Each instruction must be clear and unambiguous.
  4. Finiteness: Must terminate after a finite number of steps.
  5. Effectiveness: Every step must be basic enough to be executed manually with paper/pencil in finite time.

(b) Discuss the different Asymptotic notations and their importance.

  • Big-O (\(O\)): Upper bound of complexity. Guarantees that the algorithm runs no slower than this.
  • Big-Omega (\(\Omega\)): Lower bound of complexity. Guarantees the algorithm takes at least this long.
  • Big-Theta (\(\Theta\)): Tight bound. Used when upper and lower bounds grow at the same rate.
  • Little-o (\(o\)): Loose upper bound (strictly less than, e.g. \(f(n) = o(g(n))\) means \(f(n)\) grows much slower than \(g(n)\)).
  • Little-omega (\(\omega\)): Loose lower bound (strictly greater than).

(c) Write the recurrence relation for computing the time complexity of "Towers of Hanoi" problem. Also solve the recurrence relation to get the time complexity.

Recurrence Relation: Moving \(n\) disks from source peg to destination peg requires moving \(n-1\) disks to an auxiliary peg, moving the largest disk to the destination, and then moving the \(n-1\) disks from auxiliary to destination.

\(T(n) = 2T(n-1) + 1, \quad T(1) = 1\)

Solving by Expansion:

  • \(T(n) = 2T(n-1) + 1\)
  • \(T(n) = 2[2T(n-2) + 1] + 1 = 2^2 T(n-2) + 2 + 1\)
  • \(T(n) = 2^3 T(n-3) + 2^2 + 2^1 + 2^0\)
  • For \(k\) steps: \(T(n) = 2^k T(n-k) + (2^{k-1} + 2^{k-2} + \dots + 2^0)\)
  • Since base case is at \(n-k = 1 \implies k = n-1\): \[T(n) = 2^{n-1} T(1) + \sum_{i=0}^{n-2} 2^i = 2^{n-1} + (2^{n-1} - 1) = 2^n - 1\]
Time Complexity: \(\Theta(2^n)\)

9. (a) Write the greedy algorithm for job sequencing with deadline.

JobSequencing(Jobs, n):
    1. Sort all jobs in descending order of their profit values.
    2. Find the maximum deadline value among all jobs. Let it be Max_D.
    3. Create a result array slot[1..Max_D] and initialize all slots to Free.
    4. For i = 1 to n:
          For j = min(Max_D, Jobs[i].deadline) down to 1:
             If slot[j] is Free:
                slot[j] = Jobs[i] // Assign job to slot j
                Break // Move to next job
    5. Return slot

(b) Find optimal solution to job sequencing where n=4, profits (100,10,5,27) and deadlines (2,1,2,1).

Jobs: \(J_1(p=100, d=2)\), \(J_2(p=10, d=1)\), \(J_3(p=5, d=2)\), \(J_4(p=27, d=1)\).

  1. Sort by profits: \(J_1(100) \to J_4(27) \to J_2(10) \to J_3(5)\).
  2. Max deadline = 2. Create slots: Slot 1 (0-1) and Slot 2 (1-2).
  3. Process \(J_1\): deadline = 2. Assign to Slot 2 (1-2).
  4. Process \(J_4\): deadline = 1. Assign to Slot 1 (0-1).
  5. Process \(J_2\): deadline = 1. Slot 1 is full. Slot 2 is full. Reject.
  6. Process \(J_3\): deadline = 2. Both slots full. Reject.
Optimal Job Schedule: {J4, J1}
Total Profit: 27 + 100 = 127

A spanning tree is an acyclic, connected subgraph of a connected, undirected graph \(G=(V, E)\) that connects all the virtues \(V\) of the graph using exactly \(|V| - 1\) edges.

10. (a) Write and explain the classes P, NP, NP-hard and NP-complete.

  • P: Decision problems solvable in polynomial time by a deterministic Turing machine.
  • NP: Decision problems verifiable in polynomial time by a deterministic Turing machine (or solvable in polynomial time by a non-deterministic Turing machine).
  • NP-Hard: Problems that are at least as hard as the hardest problems in NP. They do not have to be decision problems, nor do they have to be in NP.
  • NP-Complete (NPC): Problems that are both in NP and are NP-Hard. They represent the absolute hardest problems in NP.

(b) What is reducibility?

Reducibility is a method of converting problem \(A\) into problem \(B\). If \(A \le_p B\) (polynomial-time reducible), then any algorithm that solves \(B\) in polynomial time can be used to solve \(A\) in polynomial time by transforming the input first, showing that \(B\) is at least as hard as \(A\).

(c) "If any NP-complete problem can be solved in polynomial time then P=NP" explain.

By definition, every problem \(X \in NP\) can be reduced to an NP-Complete problem \(L\) in polynomial time. If \(L\) is solvable in polynomial time, then any problem \(X \in NP\) can be solved by first reducing it to \(L\) (poly-time) and then solving \(L\) (poly-time). Since both steps take polynomial time, \(X\) is solved in polynomial time, proving \(NP \subseteq P \implies P = NP\).

11. (a) What is a Flow Network? What are the constraints of flow in a network?

A Flow Network is a directed graph \(G=(V, E)\) where each edge \((u, v)\) has a non-negative capacity \(c(u, v) \ge 0\), with a designated source node \(S\) and a sink node \(T\).

Constraints:

  1. Capacity Constraint: The flow along any edge cannot exceed its capacity: \[0 \le f(u, v) \le c(u, v)\]
  2. Flow Conservation: The total flow entering any intermediate vertex must equal the total flow leaving it: \[\sum_{u \in V} f(u, v) = \sum_{w \in V} f(v, w) \quad \text{for all } v \in V - \{S, T\}\]
  3. Skew Symmetry: Flow in one direction is the negative of flow in the reverse direction: \[f(u, v) = -f(v, u)\]

(b) What is an augmenting path in a flow network? Explain residual capacity - how it is calculated.

An **augmenting path** is a simple path from source \(S\) to sink \(T\) in the **residual graph** \(G_f\) where each edge along the path has positive residual capacity.

The **residual capacity** \(c_f(u, v)\) is the remaining capacity on an edge that can support additional flow. It is calculated as:

For forward edges: \(c_f(u, v) = c(u, v) - f(u, v)\)
For reverse edges: \(c_f(v, u) = f(u, v)\)

(c) Write the Ford-Fulkerson Algorithm for finding maximum flow.

Ford-Fulkerson(G, S, T):
    1. Initialize flow f(u, v) = 0 for all edges.
    2. While there exists an augmenting path P in the residual graph G_f:
          a. Find the bottleneck capacity c_f(P) = min { c_f(u, v) : (u, v) in P }
          b. For each edge (u, v) in P:
                If (u, v) is a forward edge:
                   f(u, v) = f(u, v) + c_f(P)
                Else:
                   f(v, u) = f(v, u) - c_f(P)
    3. Return max flow value
