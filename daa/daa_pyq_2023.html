Group A — Very Short Answer Type Questions

1. (i) State True/False: Approximation algorithm guarantees the best quality of solution.

Answer: False. An approximation algorithm guarantees a solution within a bounded ratio (approximation ratio) of the optimal solution, but not necessarily the absolute optimal (best) solution itself.

1. (ii) State True/False: O(log n) > O(n).

Answer: False. The logarithmic complexity \(O(\log n)\) grows strictly slower than linear complexity \(O(n)\), meaning \(O(\log n) < O(n)\) for all large \(n\).

1. (iii) What is the time complexity of Knapsack algorithm using Greedy method?

Answer: \(O(n \log n)\) (for the Fractional Knapsack problem, because sorting the items by profit-to-weight ratio takes \(O(n \log n)\), followed by a linear scan of \(O(n)\)).

1. (iv) State True/False: Adjacency matrix and Path Matrix are same.

Answer: False. An adjacency matrix shows direct single-edge connections between vertices, whereas a path matrix (transitive closure) shows whether a path of any length exists between vertices.

1. (v) State True/False: Ordered searching algorithm is a Polynomial Algorithm.

Answer: True. An ordered search (like Binary Search) runs in \(O(\log n)\) time, which is logarithmic (and therefore polynomial/sub-polynomial) and highly efficient.

1. (vi) State True/False: Brute Force method gives the best quality of solution.

Answer: True. Because brute force evaluates all candidate solutions in the search space, it is guaranteed to find the absolute optimal (best) solution, despite its high time complexity.

1. (vii) State True/False: Travelling Salesman Problem is a Non-Polynomial problem.

Answer: True. The TSP is NP-hard, meaning no polynomial-time algorithm is currently known to solve it.

1. (viii) Name one heuristic method of searching.

Answer: A* Search Algorithm (or Best-First Search, Hill Climbing, Genetic Algorithms).

1. (ix) Best case time complexity of Binary search algorithm is:

Answer: \(O(1)\) (occurs when the middle element of the array is the target on the first comparison).

1. (x) The time complexity of Floyd's algorithm is:

Answer: \(O(V^3)\).

1. (xi) Backtracking follows ______ traversal technique.

Answer: Depth-First Search (DFS) (using recursive stack frames to explore and backtrack).

1. (xii) What is spanning tree?

Answer: A spanning tree is a connected, acyclic subgraph of a connected, undirected graph \(G = (V, E)\) that includes all the vertices \(V\) of the graph with exactly \(|V| - 1\) edges.

Group B — Short Answer Type Questions

2. Write down an algorithm to solve the Job Sequencing with Deadline Problem using Greedy method. What is the time complexity of your algorithm?

Algorithm:

JobSequencing(Jobs, n):
    1. Sort all jobs in descending order of profit.
    2. Find max_deadline among all jobs.
    3. Create slot array slot[1..max_deadline] initialized to Free.
    4. For i = 1 to n:
          For j = min(max_deadline, Jobs[i].deadline) down to 1:
             If slot[j] is Free:
                slot[j] = Jobs[i] // Assign job
                Break
    5. Return slot

Complexity Analysis: Sorting takes \(O(n \log n)\). The nested search loops take \(O(n \cdot \min(n, \text{max\_deadline}))\). Thus, the worst-case time complexity is \(O(n^2)\).

3. Find the maximum flow of the following network. Mention each step clearly.

Given capacity bounds: \(1 \to 4 (27)\), \(1 \to 2 (24)\), \(2 \to 4 (6)\), \(2 \to 3 (10)\), \(2 \to 6 (8)\), \(4 \to 5 (18)\), \(4 \to 3 (15)\), \(5 \to 6 (18)\), \(3 \to 6 (8)\).

We trace augmenting paths from source 1 to sink 6:

  1. Path 1: \(1 \to 4 \to 5 \to 6\)
    Bottleneck capacity = \(\min(27, 18, 18) = 18\).
    Flow along Path 1 = 18. Residual capacities: \(1\to4: 9\), \(4\to5: 0\), \(5\to6: 0\).
  2. Path 2: \(1 \to 2 \to 6\)
    Bottleneck capacity = \(\min(24, 8) = 8\).
    Flow along Path 2 = 8. Residual capacities: \(1\to2: 16\), \(2\to6: 0\).
  3. Path 3: \(1 \to 2 \to 3 \to 6\)
    Bottleneck capacity = \(\min(16, 10, 8) = 8\).
    Flow along Path 3 = 8. Residual capacities: \(1\to2: 8\), \(2\to3: 2\), \(3\to6: 0\).

Since all edges directly entering sink 6 (\(5 \to 6\), \(2 \to 6\), and \(3 \to 6\)) have residual capacity 0, no more augmenting paths exist. The maximum flow is the sum of flows:

Maximum Flow: 18 + 8 + 8 = 34

4. Write down an algorithm in Greedy method to find the minimum spanning tree by Prim's algorithm. What is the time complexity of your algorithm?

Prim-MST(G, w, r):
    1. For each u in G.V:
          u.key = Infinity, u.parent = NIL
    2. r.key = 0
    3. Initialize Min-Priority Queue Q containing all vertices V.
    4. While Q is not empty:
          u = Extract-Min(Q)
          For each v in G.adj[u]:
             If v is in Q and w(u, v) < v.key:
                v.parent = u
                v.key = w(u, v)
                Decrease-Key(Q, v, w(u, v))

Complexity Analysis: With a Binary Heap, extracting minimum takes \(O(V \log V)\) and updating key values takes \(O(E \log V)\), yielding a total time complexity of \(O(E \log V)\). With a Fibonacci Heap, it runs in \(O(E + V \log V)\).

5. (a) State Cook's Theorem. (b) Define Clique with example.

(a) Cook's Theorem: States that the **Boolean Satisfiability Problem (SAT)** is NP-Complete. It was the first problem proven to belong to the NP-Complete class, providing a foundation for proving other problems are NP-Complete via polynomial-time reductions.

(b) Clique: A clique in an undirected graph \(G=(V, E)\) is a subset of vertices \(V' \subseteq V\) such that every pair of vertices in \(V'\) is connected by a direct edge (creating a complete subgraph).

Example: In a graph with vertices \(\{A, B, C, D\}\) and edges \(\{(A,B), (B,C), (C,A), (C,D)\}\), the subset \(\{A, B, C\}\) forms a clique of size 3 because edges exist between all three pairs.

6. Define the following terms: (i) Brute Force Algorithm, (ii) Approximation Algorithm.

(i) Brute Force Algorithm: A straightforward, exhaustive algorithm design strategy that checks every possible candidate solution in the search space to find the optimal one. It guarantees correctness but is highly inefficient (often taking exponential or factorial time).

(ii) Approximation Algorithm: An efficient (polynomial-time) algorithm used to find a near-optimal solution for NP-Hard optimization problems. It guarantees that the computed solution's quality is within a provable ratio (approximation ratio) of the true optimal solution.

(b) Write an algorithm to find all solutions of N-Queens problem using backtracking.

SolveNQueens(row, N, board):
    If row == N:
       Print/Store board state (Solution found)
       Return
    For col = 0 to N-1:
       If IsSafe(row, col, board, N):
          board[row] = col // Place Queen
          SolveNQueens(row + 1, N, board) // Recur
          board[row] = -1 // Backtrack

(c) What is the time complexity of your algorithm?

The time complexity is \(O(N!)\), because there are \(N\) choices for the first row, at most \(N-2\) for the second, and so on.

(d) Write an algorithm to find all pairs shortest path using Floyd's method.

Floyd-Warshall(W, V):
    D^(0) = W
    For k = 1 to V:
       For i = 1 to V:
          For j = 1 to V:
             D^(k)[i][j] = min( D^(k-1)[i][j], D^(k-1)[i][k] + D^(k-1)[k][j] )
    Return D^(V)

8. (a) Write down an algorithm of Quick Sort. (b) Derive the best, worst and average case time complexity of your algorithm.

(a) Quick Sort Algorithm:

QuickSort(A, low, high):
    If low < high:
       p = Partition(A, low, high)
       QuickSort(A, low, p - 1)
       QuickSort(A, p + 1, high)

Partition(A, low, high):
    pivot = A[high]
    i = low - 1
    For j = low to high - 1:
       If A[j] <= pivot:
          i = i + 1
          Swap A[i] with A[j]
    Swap A[i+1] with A[high]
    Return i + 1

(b) Derivations:

  • Best Case (\(O(n \log n)\)): Pivot splits array into equal halves. \(T(n) = 2T(n/2) + cn \implies T(n) = \Theta(n \log n)\).
  • Worst Case (\(O(n^2)\)): Array is already sorted and pivot is at end. \(T(n) = T(n-1) + cn \implies T(n) = \Theta(n^2)\).
  • Average Case (\(O(n \log n)\)): Assuming random splits, the expected depth of the recursion tree remains \(O(\log n)\). Writing the average case recurrence: \[T(n) = \frac{2}{n} \sum_{i=0}^{n-1} T(i) + cn \implies T(n) = O(n \log n)\]

(b) Write down an algorithm using Greedy method to find minimum spanning tree by Kruskal's algorithm. (c) Find the time complexity.

Kruskal-MST(G, w):
    1. Initialize set A = Empty
    2. For each vertex v in G.V:
          Make-Set(v)
    3. Sort the edges of G.E in non-decreasing order of weights w.
    4. For each edge (u, v) in G.E (sorted):
          If Find-Set(u) != Find-Set(v):
             Add (u, v) to A
             Union(u, v)
    5. Return A

Time Complexity: Sorting edges takes \(O(E \log E)\). Disjoint-set operations take \(O(E \cdot \alpha(V))\). Total complexity is \(O(E \log E)\) or \(O(E \log V)\).

(d) Differentiate between Backtracking and Branch and Bound.

Backtracking uses DFS search; branch and bound uses BFS/Best-First search. Backtracking is typically used for constraint-satisfaction/decision problems; branch and bound is used for optimization problems. Backtracking has lower memory requirements (\(O(V)\)); branch and bound has very high memory overhead.

10. (a) Find the transitive closure of vertex A from the following graph.

Given directed edges: \(A \to B, A \to C, B \to E, C \to D, D \to H, H \to G, G \to E, E \to F, F \to A, F \to G\).

We trace all vertices reachable from A using Depth-First Search (DFS) starting at node A:

  1. Start at A.
  2. Traverse \(A \to B\). Reach B.
  3. Traverse \(B \to E\). Reach E.
  4. Traverse \(E \to F\). Reach F.
  5. Traverse \(F \to G\). Reach G.
  6. From G, can go to E (visited). From F, can go to A (visited).
  7. Backtrack to A. Traverse \(A \to C\). Reach C.
  8. Traverse \(C \to D\). Reach D.
  9. Traverse \(D \to H\). Reach H.
  10. Traverse \(H \to G\). G is already reached.

All vertices \(\{A, B, C, D, E, F, G, H\}\) are successfully reached.

Transitive Closure of Vertex A: {A, B, C, D, E, F, G, H}

(b) Define graph terms.

  • Directed graph: A graph where edges have a defined direction (ordered pairs of vertices).
  • Undirected graph: A graph where edges connect vertices symmetrically (unordered pairs).
  • In-Degree: Number of edges directed into a vertex. Out-Degree: Number of edges directed out of a vertex.
  • Complete Graph: A graph where every pair of vertices is connected by a unique edge.
  • Articulation Point: A vertex whose removal increases the number of connected components in the graph.
  • Bridge: An edge whose removal increases the number of connected components in the graph.

11. (a) Differentiate between recursion and iteration. (b) Solve recurrence relations. (c) Define little omega notation.

(a) Recursion vs Iteration: Recursion solves problems by calling the same function on smaller inputs (uses function stack frame overhead). Iteration uses loops (for/while) to run code repeatedly (constant stack space, generally faster).

(b) Recurrence Solvers:

(i) \(T(n) = 2T(n/4) + n^{0.51}\)
\(a=2, b=4 \implies n^{\log_4 2} = n^{0.5}\).
Since \(f(n) = n^{0.51} = \Omega(n^{0.5 + 0.01})\), and the regularity condition holds (\(2(n/4)^{0.51} \approx 0.986 n^{0.51} \le c n^{0.51}\) with \(c=0.986 < 1\)), this matches Case 3 of Master's Theorem.
Complexity: \(T(n) = \Theta(n^{0.51})\).

(ii) \(T(n) = \sqrt{2} T(n/2) + \log n\)
\(a=\sqrt{2}, b=2 \implies n^{\log_2 \sqrt{2}} = n^{0.5}\).
Since \(f(n) = \log n = O(n^{0.5 - 0.1})\) (grows strictly slower than \(n^{0.5}\)), this matches Case 1 of Master's Theorem.
Complexity: \(T(n) = \Theta(n^{0.5}) = \Theta(\sqrt{n})\).

(c) Little Omega (\(\omega\)) Notation:

Indicates a loose lower bound. Formally, \(f(n) = \omega(g(n))\) if for any positive constant \(c > 0\), there exists \(n_0\) such that \(f(n) > c \cdot g(n) \ge 0\) for all \(n \ge n_0\). This implies \(\lim_{n \to \infty} \frac{f(n)}{g(n)} = \infty\).

Example: \(f(n) = n^2 = \omega(n)\) since \(\lim_{n \to \infty} \frac{n^2}{n} = \lim_{n \to \infty} n = \infty\).
