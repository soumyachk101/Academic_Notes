1. Automata Basics & Finite State Machines

1.1 Define alphabet, string and language. Difference between finite and infinite languages? Limitations of sequential circuit? Applications of Grammars.

Alphabet (Σ): A finite, non-empty set of symbols. Ex: Σ={0,1}.
String: A finite sequence of symbols chosen from an alphabet. Ex: "010".
Language (L): A set of strings formed from Σ*. Ex: L={0, 00, 000}.

Finite vs Infinite Language: Finite languages have a countable, limited number of strings (e.g., L={"a", "aa"}) and are always regular. Infinite languages have an unbounded number of strings (e.g., L={anbn}) and can be regular or non-regular.

Limitations of Sequential Circuits: They have a strictly finite amount of memory (states). They cannot process languages that require unbound memory (like counting matching parentheses to infinity).

Applications of Grammars: Used heavily in compiler design (syntax analysis/parsing), natural language processing, and defining the structure of document formats like XML/HTML.

1.2 Mealy vs Moore Machine. What is the basic difference?

Moore Machine: The output depends only on the present state. Outputs are associated with states.
Mealy Machine: The output depends on both the present state and the current input. Outputs are associated with transitions (edges).

1.3 Convert Mealy to Moore / Moore to Mealy concept.

Mealy to Moore: For every state in the Mealy machine, observe the incoming transitions. If a state has incoming transitions with different outputs (e.g., one arrow outputs 0, another outputs 1), split that state into two (e.g., q0_0 and q0_1), associating the specific output to the new state.

Moore to Mealy: Take the output associated with each state and shift it onto all the incoming transition edges directed towards that state.

1.4 Sequence detector for 0101 (overlapping).

States: q0 (Initial), q1 (Saw 0), q2 (Saw 01), q3 (Saw 010), q4 (Saw 0101).
Transitions:
- q0: on 0 → q1, on 1 → q0
- q1: on 0 → q1, on 1 → q2
- q2: on 0 → q3, on 1 → q0
- q3: on 0 → q1, on 1 → q4 (Output = 1)
- q4 (overlapping): on 0 → q3 (since the last '010' is part of the next sequence), on 1 → q0.

1.5 Minimization & k-equivalent states. Definition of lossy/lossless.

k-equivalent states: Two states p and q are k-equivalent if no input string of length ≤ k can distinguish them (i.e., both lead to final states, or both lead to non-final states). If they are k-equivalent for all k, they are combined during minimization using the partition method (Myhill-Nerode).

Lossless Machine: Given the initial state, final state, and output sequence, the exact input sequence can be uniquely deduced.
Lossy Machine: Multiple distinct input sequences can cause the same output and final state, making the input irrecoverable.

1.6 Objective Qs: Arden's, FSA concept, FSM recognizes.

  • Solution to equation R = Q + RP is R = QP* (Arden's Theorem).
  • The concept of FSA is heavily used in the Lexical Analysis phase of a compiler.
  • An FSM can uniquely recognize Regular Languages.
  • The set of regular languages is closed under Union, Intersection, Concatenation, Kleene Closure, and Complementation.

2. Regular Expressions & Languages

2.1 What are Kleene Closure and Positive Closure?

Kleene Closure (Σ*): The set of all possible strings of any length over Σ, including the empty string ε. Ex: {a}* = {ε, a, aa, aaa...}

Positive Closure (Σ+): The set of all possible strings of length ≥ 1. It excludes ε. Ex: {a}+ = {a, aa, aaa...}

2.2 State and prove Arden's theorem.

Statement: If P and Q are two regular expressions, and P does not contain ε, then the equation R = Q + RP has a unique solution R = QP*.

Proof: Substitute R = QP* into the RHS:
RHS = Q + (QP*)P = Q + QP*P
Factor out Q: RHS = Q(ε + P*P)
Since ε + P*P = P*, RHS = QP* = LHS. Hence proved.

2.3 State the Myhill-Nerode theorem.

A language L is regular if and only if its indistinguishability equivalence relation (RL) has a finite index (a finite number of equivalence classes). The number of equivalence classes corresponds exactly to the number of states in the minimal DFA that accepts L.

2.4 Using Pumping Lemma, prove L = {0n1n | n > 0} is not regular.

1. Assume L is regular. Let 'p' be the pumping length.
2. Choose a string s ∈ L such that s = 0p1p. Note |s| ≥ p.
3. By Pumping Lemma, s can be divided into xyz such that |xy| ≤ p, |y| > 0, and xyiz ∈ L for all i ≥ 0.
4. Since |xy| ≤ p, the substring 'y' must consist entirely of 0s (y = 0k for some k > 0).
5. Pump y (let i = 2): the new string is xy2z = 0p+k1p.
6. Since p+k ≠ p, this pumped string has more 0s than 1s, so it is NOT in L.
7. This contradicts the lemma. Hence, L is not regular.

2.5 Construct DFAs for specific languages.

Strings ending with 'aaa': 4 states (q0, q1, q2, q3).
q0-a→q1, q1-a→q2, q2-a→q3(Final). Any 'b' resets to q0. q3-a→q3, q3-b→q0.

Strings ending with '00': 3 states (q0, q1, q2).
q0-0→q1, q1-0→q2(Final). Any '1' resets to q0. q2-0→q2, q2-1→q0.

Strings ending with '001': 4 states. Tracks consecutive 0s, and if a 1 follows two 0s, it accepts.

3. Grammars (CFG, CSG, Normal Forms)

3.2 What is a production rule? What are terminals and non-terminals?

Production Rule: A rewrite rule specifying how symbols can be substituted to generate strings. (e.g., S → aSb).

Terminals: Symbols that make up the final strings of the language (e.g., 'a', 'b'). They cannot be replaced further.
Non-terminals: Placeholder variables that must be replaced using production rules to eventually yield terminals (e.g., 'S', 'A').

3.3 Compare LMD and RMD. What is Ambiguity?

Leftmost Derivation (LMD): Always replaces the leftmost non-terminal first.
Rightmost Derivation (RMD): Always replaces the rightmost non-terminal first.

Ambiguity: A grammar is ambiguous if a single string has more than one valid Leftmost Derivation (or more than one valid Parse Tree). Example: E → E+E | E*E | a is ambiguous because "a+a*a" can be parsed two ways (giving priority to + or *).

3.4 Grammar Simplification (Unit, Epsilon, Useless).

Unit Production: Rule of the form A → B. Removed by substituting B's RHS directly into A.
Epsilon (ε) Production: Rule of the form A → ε. Removed by finding all nullable variables and generating new rules simulating their absence.
Useless Symbols: 1. Non-generating: Variables that never yield a terminal string. 2. Non-reachable: Variables that can never be reached from the Start symbol. Both are deleted to form a Reduced Grammar.

3.5 Define CNF and GNF.

Chomsky Normal Form (CNF): Every rule is either A → BC (two non-terminals) or A → a (one terminal).

Greibach Normal Form (GNF): Every rule is A → aα (exactly one terminal followed by zero or more non-terminals). Prevents left-recursion.

4. Pushdown Automata (PDA)

4.1 Design a PDA for L = {anbn | n ≥ 1}.

Logic: For every 'a' read, PUSH it onto the stack. When 'b' is read, POP an 'a' from the stack. If the stack becomes empty (reaching the initial stack symbol Z0) exactly when the input string ends, ACCEPT.

Transitions:
1. δ(q0, a, Z0) = (q0, aZ0) [Push]
2. δ(q0, a, a) = (q0, aa) [Push]
3. δ(q0, b, a) = (q1, ε) [Pop]
4. δ(q1, b, a) = (q1, ε) [Pop]
5. δ(q1, ε, Z0) = (q_accept, Z0) [Accept by final state]

4.2 Difference between DPDA and NPDA.

DPDA (Deterministic): Has at most one valid transition for any given (state, input, stack-top). Accepts Deterministic CFLs.
NPDA (Non-Deterministic): Can have multiple choices (guessing). Accepts all Context-Free Languages. NPDA is strictly more powerful than DPDA (e.g., L={wwR} requires an NPDA to guess the midpoint).

4.3 Prove that CFLs are not closed under intersection.

Let L1 = {anbncm | n, m ≥ 0} (CFL, push a's, pop on b's, ignore c's).
Let L2 = {ambncn | n, m ≥ 0} (CFL, ignore a's, push b's, pop on c's).
Intersection L1 ∩ L2 = {anbncn | n ≥ 0}.
This resulting language requires matching three counts, which a single-stack PDA cannot do. It is Context-Sensitive, not Context-Free. Thus, CFLs are not closed under intersection.

5. Turing Machines & Undecidability

5.1 What is a Turing Machine? What is the Halting Problem?

Turing Machine: An abstract mathematical model with an infinite tape, a read/write head, and a state register. It can move left/right and represents the theoretical limits of computation.

Halting Problem: The problem of determining, given a program and an input, whether the program will eventually stop (halt) or run forever. Alan Turing proved this problem is undecidable—no universal algorithm can exist to solve it for all possible inputs.

5.2 Multi-Tape and Multi-Head TM. Are they more powerful?

Multi-Tape TM: Has multiple independent tapes and heads.
Multi-Head TM: Has one tape but multiple read/write heads.
Power: They are faster for certain algorithms, but they are not more computationally powerful than a standard single-tape TM. Anything a multi-tape TM can compute, a single-tape TM can simulate.

5.3 Explain Church's Hypothesis and Universal Turing Machine.

Church-Turing Thesis: A hypothesis stating that any function that is intuitively computable by a human following an algorithm can be computed by a Turing Machine.

Universal Turing Machine (UTM): A TM that takes the encoded description of any other TM 'M' and its input 'w' on its tape, and perfectly simulates 'M' running on 'w'. It is the theoretical foundation of programmable computers (software).

5.4 Design a TM that accepts 0n1n.

Logic:
1. Read '0', replace it with 'X', move Right.
2. Skip all '0's and 'Y's until a '1' is found.
3. Replace '1' with 'Y', move Left.
4. Skip all 'Y's and '0's backwards until 'X' is found, move Right.
5. Repeat until all '0's are 'X's and '1's are 'Y's. If no unmarked 0s or 1s remain, ACCEPT.
